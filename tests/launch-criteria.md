# よるせん ローンチ基準

## ✅ ローンチOKの定義

以下の全項目をクリアしたらローンチ可能。

---

## 1. 機能テスト（自動）- 全部PASS必須

| テスト | 基準 |
|--------|------|
| オンボーディング完了 | 全画面遷移できる・エラーなし |
| AIチャット | 質問送信→ストリーミング返答が来る |
| お客さん追加 | 登録→一覧に表示される |
| 来店記録 | 追加→単価が更新される |
| LINE文案生成 | ボタン押下→テキストが生成される |
| カレンダー | 日付タップ→来店記録モーダル開く |
| 売上分析 | 月別・顧客別タブが表示される |
| コース登録 | 追加→来店フォームに反映される |
| プレミアムページ | Stripeページへ遷移する（審査後） |
| 保存済み | 保存→一覧に表示される |

## 3. UX評価（ヒューリスティック）- 手動確認

Nielsenの10原則に基づく評価。各項目を 1〜5 で採点。平均3.5以上でローンチOK。

| 原則 | チェック内容 |
|------|------------|
| 1. 状態の可視化 | 処理中・完了・エラーが画面で分かる |
| 2. 現実との対応 | 夜職用語が正しく使われている |
| 3. ユーザーコントロール | 削除・キャンセル・戻るが使える |
| 4. 一貫性 | ボタン・色・フォントが統一されている |
| 5. エラー防止 | 空欄で保存できない等 |
| 6. 認識負荷の軽減 | アイコン+ラベルで意味が分かる |
| 7. 柔軟性 | 初心者も上級者も使いやすい |
| 8. 審美的なデザイン | 夜職女子に刺さるUI |
| 9. エラー回復 | エラーメッセージが分かりやすい |
| 10. ヘルプ | 迷ったときの説明がある |
